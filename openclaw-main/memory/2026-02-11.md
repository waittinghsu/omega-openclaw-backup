## 備份架構大改造

### 問題
用戶指出：未來會有多個 OpenClaw 實例，需要在同一個 repo 分開備份

### 解決方案
重組為多實例備份結構：

**舊架構（單實例）：**
```
~/.openclaw/workspace/
├── .git/
├── SOUL.md
└── ...
```

**新架構（多實例）：**
```
~/openclaw-backups/
└── omega-openclaw-backup/
    ├── README.md                  # 總說明
    ├── openclaw-main/             # 佛祖
    │   ├── SOUL.md
    │   ├── memory/
    │   └── ...
    └── openclaw-zeabur/           # 未來龍蝦（待建立）
```

### 實作步驟
1. 建立 `~/openclaw-backups/openclaw-main/`
2. 複製 workspace 內容到新位置
3. 撰寫詳細 README（給未來實例看的初始化指引）
4. 初始化新 git repo
5. Force push 到 GitHub（覆蓋舊結構）

### README 內容
- 📂 當前實例清單
- 🤖 新實例初始化流程（主動詢問名稱）
- 🔐 安全須知
- 📁 目錄結構範例
- 🔄 復原方式
